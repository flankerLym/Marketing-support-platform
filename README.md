# 大营销抽奖接口压测结果

## 压测说明

本次压测针对大营销项目的抽奖核心接口 `draw`，采用 JMeter 进行阶梯式并发压测与稳定性压测。压测过程中重点观察吞吐量、平均响应时间、P90/P95/P99、异常率、接收/发送流量等指标。

核心观察结论：

- 随着线程数提升，吞吐量并非线性增长。
- 第一轮压测中，线程数达到 300 以后，响应时间明显升高，继续加线程收益下降。
- 第二轮缩小压测区间后，发现 200～250 线程附近是较优区间。
- 调整数据库连接池、Redis 连接池等参数后，第三轮在 200 线程下进行稳定性压测，40 万请求全部成功，吞吐达到 607.8 TPS。

### 第三轮结论

第三轮是最终稳定性结果。在 200 并发线程下，抽奖接口完成 40 万次请求，错误率 0.00%，稳定吞吐达到 607.8 TPS，平均响应时间 318ms，P95 为 700ms，P99 为 1181ms。相比第二轮同为 200 线程时的 516.2/sec，调优后吞吐提升至 607.8/sec，说明连接池、日志降噪和运行参数优化有效。

---

## 总体结论

本次压测表明，抽奖接口在当前环境下的较优并发点为 200 线程左右。低并发阶段吞吐量随线程数提升而增长；当线程数继续增加到 300 以上时，响应时间和 P99 明显升高，吞吐量不再线性增长，说明系统开始进入连接池、线程池或下游资源竞争区间。

最终稳定性压测结果为：

- 稳定并发：200 线程
- 总请求数：400000
- 错误率：0.00%
- 平均响应时间：318ms
- P95：700ms
- P99：1181ms
- 稳定吞吐量：607.8 TPS
